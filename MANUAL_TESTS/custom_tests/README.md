# Custom Tests Directory

This directory contains custom test files generated by the Live Configuration Editor.

## How to Create Custom Tests

1. Open the Enhanced Test Launcher (`make enhanced-tests` or `python MANUAL_TESTS/enhanced_test_launcher_v2.py`)
2. Go to the "⚙️ Configuration Editor" tab
3. Configure your test parameters:
   - Grid size, agent count, resource density
   - Perception radius, distance scaling factor
   - Preference mix and seed
4. Click "💾 Save Custom Test" to save your configuration as a test file
5. The test will appear in the "🔧 Custom Tests" tab of the Enhanced Test Launcher

## Generated Test Files

Each custom test file contains:
- Complete test configuration based on your parameters
- Executable Python script that can be run standalone
- Metadata about creation date and configuration
- Integration with the standard test framework

## Running Custom Tests

### From Enhanced Test Launcher
- Use the "🔧 Custom Tests" tab
- Click "🚀 Launch" on any custom test card
- Click "✏️ Edit" to modify the test file

### From Command Line
```bash
# Navigate to the custom_tests directory
cd MANUAL_TESTS/custom_tests/

# Run any custom test directly
python your_custom_test.py
```

## File Format

Custom test files follow this structure:
```python
#!/usr/bin/env python3
"""
Custom Generated Test: [Your Test Name]
[Your Description]

Auto-generated by Live Configuration Editor  
Created: [Timestamp]
"""

# Standard imports and test configuration
CUSTOM_CONFIG = TestConfiguration(
    # Your parameters here
)

class CustomTest(StandardPhaseTest):
    def __init__(self):
        super().__init__(CUSTOM_CONFIG)

# Main execution function
```

## Tips

- Use descriptive names for your custom tests
- The Enhanced Test Launcher automatically discovers new test files
- Test files are immediately executable after creation
- You can edit test files manually if needed
- Custom tests integrate with the standard phase management system
